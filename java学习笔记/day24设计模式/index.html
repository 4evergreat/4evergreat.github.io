1.代理

Java代理是一种设计模式，它允许在不改变原始类或接口的情况下对其进行增强或修改。在运行时通过创建一个代理对象来控制对原始对象的访问，并允许我们在访问原始对象之前或之后执行一些额外的逻辑，从而实现一些特定的功能，如日志记录、安全控制、缓存处理、远程调用、延迟加载和事务处理等。
